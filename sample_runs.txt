# RIGOROUS BOUNDARY CASE REPORT - COSMOS SEARCH ENGINE

This report documents the end-to-end correctness of the search pipeline against verified boundary conditions.

## 1. Minimal Input Cases

### Empty Query
- **Condition**: User submits a blank search box.
- **Expected Behavior**: Return empty result set with a 200 OK status and a "Empty query" message.
- **Actual Status**: [PASS] - Correctly identified in `app.py` and handled before index traversal.

### Stop-Words Only
- **Condition**: Query contains only common words (e.g., "the and in").
- **Expected Behavior**: After tokenization, the query token list should be empty. Return empty list.
- **Actual Status**: [PASS] - `SearchEngine` filters these out, returning `[]` immediately.

---

## 2. Term Match Cases

### No Matches (Zero-Hit)
- **Condition**: Query for a term that does not exist in any document (e.g., "xenomorph").
- **Expected Behavior**: Compressed Trie lookup returns -1. Result is an empty list.
- **Actual Status**: [PASS] - System returns 0 results as expected.

### Partial Matches (Prefix)
- **Condition**: Query for a word fragment (e.g., "rocke" for "rocket").
- **Expected Behavior**: In a standard keyword search, this should return no matches unless a prefix-expand algorithm is used. 
- **Actual Status**: [PASS - Exact Match Only] - As per textbook Inverted Index design, the engine currently requires full-term matches. Future expansion to `search_prefix` is implemented in the Trie but disabled in the main API for strictness.

---

## 3. Multi-Keyword Logic (AND)

### Full Intersection
- **Condition**: Query "Mars Starship" (both terms appear on some pages).
- **Expected Behavior**: Return only pages containing BOTH keywords. Rank by combined frequency.
- **Actual Status**: [PASS] - Merge algorithm correctly identifies `mars_colonization.html` as the top hit.

### No Intersection
- **Condition**: Query "Moon Voyager" (both exist, but not necessarily on all same pages).
- **Actual Status**: [PASS] - Results correctly show `index.html` (which links both) but NOT pages that only contain one of the two.

---

## 4. Normalization Robustness

### Punctuation & Case
- **Condition**: Query "mOOn!!!" 
- **Expected Behavior**: Identical results to "moon".
- **Actual Status**: [PASS] - Normalization pipeline strips '!' and downcases the search.

### Excessive Whitespace
- **Condition**: Query "     rocket     "
- **Expected Behavior**: Single term "rocket" extraction.
- **Actual Status**: [PASS] - `split()` gracefully handles variable whitespace.
